<Note>
  Relay is available at `/relay`. The legacy `/connect` route redirects to Relay.
</Note>

The Relay home renders `ConnectOverviewPage`. When the active home view is open, it renders `ConnectDashboard`.

## Steps

<Steps>
  <Step title="Open Relay">
    Go to `/relay`. If you use the older `/connect` link, LifelineIQ redirects you to Relay.
  </Step>
  <Step title="Check your duty status">
    The clock action bar shows your current state for today's time entry.

    If there is no time entry for today with `clock_in`, and there is no `clock_out`, the bar shows **Off Duty** with a **Clock In** action.

    If `todayEntry.clock_in` exists and `todayEntry.clock_out` is empty, the bar shows **On Duty** with **Shift Time** and a **Clock Out** action.
  </Step>
  <Step title="Clock in">
    Select **Clock In** while the action bar shows **Off Duty**.

    The app sends a `POST` request to `/api/personnel/time-entries/clock-in` with `user_id`, `tenant_id`, `agency_id`, `entry_date`, `clock_in`, and `status: pending`.

    On success, LifelineIQ shows a toast titled **Clocked in** with the description **You have successfully clocked in.**
  </Step>
  <Step title="Clock out">
    Select **Clock Out** while the action bar shows **On Duty**.

    The app sends a `PATCH` request to `/api/personnel/time-entries/{todayEntry.id}/clock-out`.
  </Step>
</Steps>

## What happens next

When you clock in, the backend creates a new `personnel.time_entries` row. The OpenAPI contract records `clock_in=NOW()`, `clock_out=NULL`, and `breaks=[]`.

When you clock out, the backend sets `clock_out=NOW()`. If there is an open break on the time entry, the backend closes that break too.

## Common problems

**Clock In is not available**

The action bar only shows **Clock In** when today's entry is not already open. If `todayEntry.clock_in` exists and `todayEntry.clock_out` is empty, the bar shows **On Duty** and **Clock Out** instead.

**Clock-in returns a conflict**

The backend returns `409` when the user already has an open time entry. Refresh Relay and check whether the action bar now shows **On Duty**.

**Clock Out is not available**

The action bar only shows **Clock Out** when today's entry has `clock_in` and does not have `clock_out`.

## Manager follow-up

Managers should review pending time entries in the appropriate personnel timekeeping workflow. A clock-in creates the entry with `status: pending`, so the shift still needs normal review before it is finalized.
