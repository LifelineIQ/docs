Use **Automation > Workflows** for workflow automation. Use this for operational automations that need triggers, actions, schedules, templates, run history, or policy guardrails.

## Steps

<Steps>
  <Step title="Open Workflows">
    Go to **Automation > Workflows**.
  </Step>
  <Step title="Create a workflow">
    Select the action to create a new workflow, or open **Templates** if you want to start from an existing workflow template.
  </Step>
  <Step title="Build the workflow">
    In the workflow studio, define the trigger, workflow steps, conditions, and actions.
  </Step>
  <Step title="Save the workflow">
    Select the save action in the workflow studio.
  </Step>
  <Step title="Review runs">
    Open the workflow's run history when you need to check whether the workflow ran and what happened.
  </Step>
</Steps>

## Common problems

### I opened an old automation admin link

Older automation and insight admin links redirect to the current Automation or Intelligence areas. Use **Automation > Workflows** for workflow creation.

### I should not turn on a workflow yet

Save the workflow as a draft or keep it inactive until the owner has reviewed the trigger, actions, and expected effects.
